网络工程基础知识核心笔记

从 IP 地址到 TCP/UDP，一份面向入门的网络工程核心知识梳理，适合复习与面试准备。

IP 地址、子网掩码与子网划分

IP 地址

IP 地址是网络中设备的唯一标识，如同网购的精准收货地址，确保数据包在全球几十亿设备中准确找到目标设备（手机、电脑、服务器等）。

分为两种类型：

  • 公网 IP：由运营商分配，全球唯一的”门牌号”，整个互联网都可识别。
  • 私网 IP：通常以 192.168 开头，相当于局域网内的”房间号”，仅在局域网内部有效。

子网掩码

子网掩码是 IP 地址的”分隔符”，用于区分地址中的网络位和主机位。

最常见的子网掩码是 255.255.255.0，其含义是：IP 地址的前三段为网络位，最后一段为主机位。

子网划分

子网划分是将大网段拆分为多个互相隔离的小网段（子网）的过程。

例如，192.168.1.0 网段可容纳 254 台设备。若所有设备共处同一网段，易导致广播泛滥，引发网络卡顿和安全问题。通过子网划分，每个子网独立管理设备、互不干扰，既能提升网络效率，又能增强安全性。

上网必备的三大协议

ARP 协议

所有通信设备都有全球唯一的 MAC 地址，数据包在局域网内最终通过 MAC 地址定位设备。ARP 协议的作用是通过 IP 地址查询对应设备的 MAC 地址，是内网设备通信的基础。

示例： 电脑需向同局域网的打印机（IP 为 192.168.1.10）发送文件时，会在局域网内广播询问——

“谁是 192.168.1.10，请回复你的 MAC 地址”

打印机收到后回复自身 MAC 地址，电脑将其缓存，后续通信可直接使用该 MAC 地址，无需重复广播。

DHCP 协议

DHCP 协议相当于“自动分配地址的物业”，能自动为联网设备分配 IP 地址、网关、DNS 等上网参数。

  • 无需管理员手动设置
  • 自动避免 IP 地址冲突

手机连接 Wi-Fi 时，DHCP 会自动完成所有参数配置，用户无需关心底层细节。

DNS 协议

DNS 协议被称为“互联网的查询台”。

用户输入的网址（如 www.baidu.com）是人类可理解的字符，而电脑只识别数字 IP 地址。DNS 的功能就是将网址翻译成对应的 IP 地址。

没有 DNS 的世界——你得记住所有网站的数字 IP 才能上网。

交换机与路由器

交换机

局域网内的 “内部快递员”

  • 仅负责同一网段（同一”小区”）内的设备数据转发
  • 基于 MAC 地址定位设备
  • 只在局域网内工作，注重转发效率

路由器

“跨网络的全国快递分拣中心”

  • 负责跨网段、跨城市的数据包转发
  • 基于 IP 地址寻找最优路径
  • 用户能刷抖音、访问外网，都依靠路由器将数据”一跳一跳”发送到全球各地，并将结果返回

总结： 同网段内网通信靠交换机，跨网段访问互联网靠路由器。

数据传输的两大核心模式：TCP 与 UDP

TCP（传输控制协议）

核心特点：可靠优先，哪怕慢一点

工作机制：

  1. 三次握手建立连接："你准备好了吗？" → "准备好了。" → "那我开始发了。"
  2. 编号确认：发送时对每个数据包编号，接收方收到后回复确认（如”收到 1，请发 2”）
  3. 自动重传：若出现丢包，TCP 会自动重传
  4. 四次挥手断开连接：数据传输结束后优雅关闭

适用场景： 微信发文件、浏览网页、接收邮件等——这些场景不允许数据丢失。

UDP（用户数据报协议）

核心特点：快速优先，可靠性较低

工作方式可比喻为“广场喊话”——

无需握手建立连接，发送方直接发送数据，不要求接收方确认，也不进行丢包重传。

打个比方：像”三秒后有一百个包裹扔下来，你们自己接，接不到算球”。因此 UDP 速度极快但可靠性低。

适用场景： 刷抖音、实时游戏、视频会议等——对延迟敏感、偶尔卡顿可接受的场景。

速查总表

概念 关键作用 类比
IP 地址 设备唯一标识 收货地址
子网掩码 区分网络位/主机位 地址分隔符
ARP IP → MAC 地址查询 内网寻人广播
DHCP 自动分配网络参数 物业自动配钥匙
DNS 域名 → IP 解析 互联网查询台
交换机 同网段数据转发 内部快递员
路由器 跨网段数据转发 全国分拣中心
TCP 可靠传输 挂号信（有回执）
UDP 快速传输 广场喊话（不保证）
